# Systematic Workstream Record

## Frame

- Objective: Build a read-only, source-backed MapGen recipe artifact dependency
  DAG for the selected Studio recipe, served through oRPC + Effect and rendered
  as a full-screen secondary Studio tab.
- Future state: A recipe author can select a recipe, open the DAG tab, see
  phase clusters, stage nodes, artifact dependency edges, and expandable
  sequential steps without reading source files; dense recipes use a
  dependency-ranked layout that remains smooth to pan with a trackpad.
- Non-goals: Editable graph mutation APIs; runtime trace overlays; generated
  artifact edits; Civ7 runtime/direct-control changes; phase-as-order semantics.
- Hard core: Artifact edges come from authored `contract.artifacts` producer and
  consumer contracts. Recipe order remains recipe-owned. React consumes a
  prepared DTO and does not infer artifact semantics.
- Exterior: Styling overhaul, in-game proof, map rendering layers, and legacy
  generated Studio artifact formats unless the implementation proves a minimal
  contract addition is required.
- Falsifier: If authored recipe source cannot expose complete stage/step
  artifact contracts to Studio without generated-output imports or forbidden
  deep imports, this frame must be redesigned before implementation continues.
- Redesign trigger: Editable graph work needs mutation semantics that cannot
  reuse the read-only stage/step/artifact identity model.
