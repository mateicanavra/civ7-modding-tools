## Implementation

- Added package-owned direct-control `getCiv7FullMapGrid` and
  `planCiv7MapGridReadBounds` so full-grid proof reads are bounded chunks
  instead of caller-local loops.
- Added `mods/mod-swooper-maps/src/dev/diagnostics/live-parity.ts` as the
  reusable proof model:
  - validates completed exact-authorship packets, including visible source
    snapshot body fields and the visible pipeline config body hash;
  - runs local final terrain/biome/feature/resource surfaces from the
    Studio-authored pipeline config;
  - converts live direct-control grids into the same final-surface shape;
  - marks mismatches, missing live readback, runtime turn/hash/plot-count drift,
    and exact-authorship gaps as unresolved links;
  - classifies rivers/floodplains/wonders/starts residuals without treating
    them as silent proof omissions.
- Added root command:
  `bun run verify:final-surface-parity -- --request-id <id>` or
  `--proof-file <status-or-proof.json>`.
- Removed duplicate comparison logic from the root command by making it a thin
  wrapper over the reusable proof model.
- Added setup `playerCount` readback to the direct-control setup snapshot and
  taught Studio exact-authorship proof to consume that config readback when Civ
  does not expose a synthetic `PlayerCount` setup parameter row.
- Set the parity verifier's default live grid chunk size to 512 plots because a
  single 6,996-plot Huge read timed out while 17 bounded chunks completed.
