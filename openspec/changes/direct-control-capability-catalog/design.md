## Design

The catalog is a schema-backed source artifact plus generated outputs. It does
not create public high-level wrappers automatically.

## Catalog Shape

Each symbol records:

- `path`
- state role/name/phase
- kind: root, method, property, table, enum, wrapper
- access classification: read, validate, mutate, elevated, avoid
- risk classification: safe, developer, gameplay, destructive, account/network
- provenance: runtime, official resource, declaration, public corpus, wrapper
- controller method provenance when reported by the project-owned game-scoped
  controller
- confidence label
- wrapper recommendation
- observed descriptors and selected sample metadata

## Source And Generated Files

- Source schema and generator code live in `packages/civ7-direct-control/src`.
- Runtime snapshots and reference outputs live under a package-owned generated
  artifacts directory that is regenerated by script.
- Reviewed declarations for ambient Civ7 globals move into `packages/civ7-types`
  only after human/agent review records the promotion.

## TypeBox Use

Use the repo's TypeBox v1 package (`typebox`) and keep schema modules small.
Compile validators once in tests or generator entrypoints. Do not mix
`@sinclair/typebox` with `typebox`.

## Merge Strategy

- Runtime snapshot records availability and method/property names by role.
- Controller capability list records allowlisted methods, protocol version,
  bounds, and proof status for controller-backed wrappers.
- Official resource parsing supplies table and enum-like identifiers.
- Current `@civ7/types` supplies existing declaration provenance.
- Public corpus evidence may add hints but cannot raise confidence above
  `public-hint` without source or live evidence.

## Review Lanes Required

- TypeScript/API review for type-value alignment.
- Architecture review for generated/source boundaries.
- Product review for useful developer/LLM-agent reference categories.
