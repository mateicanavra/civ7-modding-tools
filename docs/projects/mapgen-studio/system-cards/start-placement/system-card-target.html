The thesis. The Civ7 engine accepts exactly one thing per player: setStartPosition(plotIndex, playerId) — ✓ civ7-adapter/src/types.ts:658. You cannot hand the engine the reasoning behind a start. So “don’t drop half the factors” is not a request to push more into the engine — it is a demand that the truth layer decide and measure everything, publish the full per-start record, and guarantee fairness before projection. The engine call stays identical; what changes is that nothing load-bearing is thrown away on the way there, and the one-way boundary is crossed only after quality and fairness are already settled and recorded.

The target algorithm, in plain language

as-built steps + the target additions, marked ◇
  1. Gather base starts & config. As today — players-per-hemisphere and the sector grid from runtime, plus the (now fully-knobbed) starts block. ✓ index.ts:30-44
  2. Validate inputs.◇ new Assert each climate/soil layer’s coverage; where a tile lacks a value, mark the affected component imputed rather than silently substituting a neutral default. ◇ closes default.ts:447-478
  3. Scan & gate. Reject water, lakes, and non-slot tiles before scoring — unchanged. ✓ default.ts:362-379
  4. Measure the land envelope & classify a tier. Continent size, expansion room, island cluster, coast distance → primary / islandCluster / marginal. Thresholds now all knobs. ✓ default.ts:208-238
  5. Score habitability — and keep the breakdown.◇ changed Compute the same six components, but retain the per-component vector on the candidate instead of collapsing straight to one scalar. The scalar is still derived for ranking; the vector survives to publication. ◇ extends default.ts:258-306
  6. Provisional selection with an honest spacing floor.◇ changed Pick & space per region as today, but the minimum-spacing floor is a published guarantee: every relaxation step is appended to fairnessReport.relaxations[], never silently swallowed. ◇ extends materialize.ts:482-530
  7. Fairness-balancing pass.◇ new Build the cross-start parity frame; if the worst-pair gap exceeds fairnessTolerance, deterministically swap the weakest marginal pick(s) for alternatives that shrink the gap — the same score-and-rebalance shape the codebase already uses for resources. Record the final parity + whether it converged. ◇ precedent: ecology/resources/score-balance
  8. Graceful degradation ladder — never throw.◇ changed regional → open-pool → quality-relaxed → spacing-relaxed. A truly unfillable map yields status:"degraded" as data (deterministically), surfaced to the caller — not an exception that aborts generation. ◇ replaces materialize.ts:214-218
  9. Last resort, still scored.◇ changed If degradation is reached, pick by a reduced-but-present quality function (not pure distance), and stamp each such start with lowered confidence in its StartRecord. ◇ replaces materialize.ts:546-576
  10. Publish the full record, then project.◇ new Emit starts[] (per-player StartRecord), fairnessReport, and status; only then stamp setStartPosition(plot, player) for each. Quality and fairness are settled before the one-way boundary. ◇ extends artifacts.ts:77-119 → materialize.ts:146-218

Target heuristics & what they buy you

Retain-then-reduce scoring

Keep the full component vector; derive the ranking scalar from it instead of discarding it.

Effect: every start’s “why” is inspectable and testable downstream — the factors are no longer dropped. closes the core gap

Measured fairness, bounded

A cross-start parity frame + a balancing swap pass with a tolerance knob.

Effect: start fairness becomes a first-class, bounded output — or its breach is published, never hidden. new capability

Degrade-as-data

An explicit ladder ending in a status:"degraded" record instead of a throw.

Effect: a hostile map yields a known, deterministic outcome the caller can react to, not a crashed generation. closes the P2 throw

Determinism is preserved — non-negotiable. Every addition is order-deterministic: the balancing swaps iterate candidates in the existing (tier, score, plotIndex) order with the same tie-breaks, the degradation ladder is a fixed sequence, and imputation is a pure function of coverage. No Math.random enters. Identical inputs + config ⇒ identical starts and an identical fairnessReport. The target buys completeness and fairness without spending reproducibility.

Acceptance criteria — a top-tier outcome passes all of these

A1

Per-start completeness

Every published StartRecord carries all six component scores + tier + plot. No factor that influenced the choice is absent from the artifact.

◇ test: starts.length === totalPlayers ∧ every record has 6 components
A2

Fairness measured & bounded

A fairnessReport is always published; either worstPairGap ≤ fairnessTolerance, or balanced:false is surfaced. Unfairness is never silent.

◇ test: report present ∧ (gap ≤ tol ∨ balanced === false)
A3

No hard failure

Assignment never throws. An unfillable map returns status:"degraded" deterministically; the caller decides what to do.

◇ test: pathological map ⇒ resolves with status:"degraded", no exception
A4

Honest spacing & scored last resort

Every spacing relaxation is recorded in relaxations[]; desperation picks use a reduced quality function and carry lowered confidence — never pure distance.

◇ test: relaxations logged ∧ desperation records have components + confidence
A5

No silent neutrality, fully tunable, still deterministic

Missing inputs are flagged imputed, every scoring constant is a knob, and identical inputs+config reproduce identical starts and fairnessReport.

◇ test: imputed flags surface ∧ re-run is byte-identical
