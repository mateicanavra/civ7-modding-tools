Physical-layer reading. A start models a habitable home region: the op fuses a loosely-physical habitability surface — land area & expansion room, fresh water, soil fertility, climate comfort, terrain roughness, nearby resources — into one scalar suitability score, then spaces the winners apart for fairness. At the engine boundary all of that collapses to a single (plotIndex, playerIndex) pair: the engine learns where each player starts but nothing about why, which tier it came from, or how close the runner-up was — so nothing downstream can rebalance against start quality.

The algorithm, in plain language

✓ verified against source
  1. Gather base starts & config. Read players-per-hemisphere and the start-sector grid from the engine mapInfo runtime, and clone the starts knob block. ✓ index.ts:30-44 · runtime via derive-placement-inputs
  2. Scan every tile, gate hard. Reject water (landMask≠1), planned lakes (lakeMask=1), and tiles outside a region slot (slot∉{1,2}) before any scoring. ✓ plan-starts/strategies/default.ts:362-379
  3. Measure the land envelope. Compute continent size, same-continent land within the expansion radius (4), any land within the island radius (5), and distance to coast. ✓ default.ts:381-408
  4. Classify a tier. single-tile → reject; primary if ≥24 contiguous & ≥14 expansion; else islandCluster if ≥18 nearby land & ≤1 from coast; else marginal at 50%/65% of the primary gates; else reject. ✓ default.ts:208-238
  5. Score habitability. Fresh water (neighbor river 0.8 / lake 0.7, on-tile river ≤0.5), resource support (placed resources within radius 4, normalized 0..1), roughness penalty (local elevation range ÷900), and radius-2 averages of fertility / moisture / aridity / temperature. ✓ default.ts:146-206, 436-478
  6. Blend into one composite score. (landScore·largeLandmassW + fertility·fertilityW + resource·resourceW + freshwater·freshwaterW + climate) ÷ (Σweights + 1), then + tierBias (±0.08/0.02) − roughness·weight·0.12, clamped 0..1. ✓ default.ts:258-306
  7. Sort candidates deterministically. By tier (primary > island > marginal), then score desc, then plotIndex asc — a stable order with no randomness. ✓ default.ts:509-514 · materialize.ts:105
  8. Select per hemisphere with spacing. Optionally filter a region's pool by active start sectors (fall back to the full region pool if empty), then greedily take the best-ranked tile at least minStartSpacing (9) from everything chosen; relax the spacing by 1 when stuck, down to 0. ✓ materialize.ts:116-144, 482-530
  9. Open-pool fallback. If a region came up short, refill the empty player slots from the combined candidate pool using the same ranked-with-spacing logic. ✓ materialize.ts:165-188
  10. Desperation fallback. If still short, pick from any remaining in-slot land tile by maximum distance only (no quality) so every player is guaranteed a tile. ✓ materialize.ts:190-212, 546-576
  11. Assert & write. Throw if not every player was placed; otherwise each chosen tile has already been stamped into the engine via adapter.setStartPosition(plot, player). ✓ materialize.ts:146-218

Key heuristics & what they buy you

Tiered viability gate

Splits land into primary / islandCluster / marginal before scoring, with hard size & coast thresholds.

Effect: protects players from tiny-island or cramped starts on continental maps, while still enabling deliberate archipelago starts. default.ts:208-238

Habitability blend, fertility-led

A weighted sum where fertility carries the top default weight (1.2); fresh water, resources, land size & climate fill in.

Effect: pulls starts toward fertile, watered, resource-adjacent, expandable land rather than raw geography. default.ts:258-306

Spacing-with-relaxation

Greedy max-spacing selection that loosens the minimum distance only when no candidate satisfies it.

Effect: spreads players apart for fairness, but degrades gracefully (toward adjacency) rather than failing on crowded maps. materialize.ts:482-530

Worked example — swooper-earthlike. Earthlike does two things at once. It loosens the viability gates (minExpansion 14→4, island cluster 18→3, coast distance 1→8), so far more tiles survive as candidates — then it quadruples largeLandmassWeight (1→4), which makes the score denominator 4 + 1.2 + 1 + 0.9 + 1 = 8.1 and lets land-envelope size alone supply ≈ 49% of a tile's base score. Net effect: cast a wide net for viable starts, then strongly prefer the most expansive interior-land starts — while resource pull is dialed back to a radius-1 halo. A clean case of a loosened gate × a dominant single weight compounding.
Determinism. Start placement uses no randomness at all — a grep of plan-starts and materialize.ts finds zero Math.random / seed / RNG / noise. Candidate order is (tier, score, plotIndex); selection ties break on (rankingScore, score, plotIndex); distances use periodic-X hex geometry. Identical upstream artifacts + identical config ⇒ identical starts, every run. ✓ default.ts:509-514 · materialize.ts:105,506-510

Issues, risks & verification notes

flagged by the team, then adversarially re-read by the owner
P2

Spacing silently relaxes to adjacency

When no candidate satisfies the minimum spacing, the selector decrements it by 1 and retries — all the way to 0. On sparse-land or high-minStartSpacingTiles maps two starts can end up a single tile apart, with no warning, telemetry, or record that the target spacing was abandoned. Spacing is a soft preference, not the guarantee its name implies.

✓ materialize.ts:491-523
P2

Assign-all-or-throw, no graceful degradation

If regional + open-pool + desperation still can't fill every player slot, the step throws and map generation fails outright. Reaching it needs fewer in-slot land tiles than players (a pathological / over-packed map), but there is no partial-placement or relaxed fallback — it is all-or-nothing. (Downgraded from a sub-agent P1: it is a fail-loud guard requiring an unrealistic config, not wrong output on a realistic one.)

✓ materialize.ts:214-218
P3

Desperation tiles ignore all quality

Once both prior tiers are exhausted, tiles are chosen by maximum spacing distance only — no fertility, fresh water, tier, or score — so a forced start can land on land the planner explicitly rejected. Gated behind an already-rare fallback, but genuinely unfair when it fires.

✓ materialize.ts:546-576
P3

Two-region bucketing regardless of continent count

Every landmass is assigned to west(1) or east(2) purely by its bounding-box center x; players are split into exactly two hemispheres even on maps with three or more major continents, and a continent straddling the midline tips to one side wholesale. By design, but load-bearing for any fairness reasoning.

✓ plot-landmass-regions/index.ts:37-38
P3

Hardcoded scoring constants without knobs

Several tuning constants are literals, not config: the roughness divisor 900 (while its weight is a knob), the tier biases +0.08 / +0.02 / −0.08, the selection blend 0.86·score + 0.14·spacing, and the marginal-tier ratios 0.5× / 0.65×. None can be tuned per preset, so the score/spacing trade-off and roughness calibration are fixed across all maps.

✓ default.ts:179, 231-235, 304-305 · materialize.ts:506
P3

Silent neutral fallbacks for missing climate/soil

When a climate/soil layer has no value for a tile, scoring substitutes fertility 0.5 / aridity 0.45 / temperature 16°C and continues. A partially-populated ecology layer biases scoring toward "neutral" rather than failing or flagging, which can quietly distort start choice.

✓ default.ts:447-478

REF

"P1 — resource-support builder has no bounds check → out-of-bounds writes / corruption"

False. buildResourceSupport coerces each index with raw | 0 and continues on plotIndex < 0 || plotIndex >= size before ever indexing the counts array. Invalid upstream indices are skipped, not written.

✓ default.ts:192-194
REF

"P2 — inactive start sectors silently starve a region, forcing the open pool"

False. The sector filter is applied as filtered.length ? filtered : regionCandidates — if it yields nothing, selection falls back to the full regional candidate pool, not an empty one. Sector misconfiguration is handled gracefully.

✓ materialize.ts:131
REF

"P3 — freshwater river/lake bonus logic is inverted in the comments"

False on two counts: there are no comments at the site, and river (0.8) is correctly weighted above lake (0.7). The scoring is internally consistent — rivers are the stronger fresh-water signal.

✓ default.ts:153-161
REF

"score = landScore·fertilityWeight + fertility·fertilityWeight + …"

Transcription error. The composite weights landScore by largeLandmassWeight and fertility by fertilityWeight — distinct weights. The corrected formula is the one used in section C, step 6.
