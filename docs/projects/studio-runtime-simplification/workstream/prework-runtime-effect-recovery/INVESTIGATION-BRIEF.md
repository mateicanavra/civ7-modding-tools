## 6. Evidence Policy

- Evidence standard: verified for file/command facts; corroborated for synthesis; unresolved for unavailable external review/merge policy.
- Source authority order:
  1. Direct current user instruction.
  2. Root `AGENTS.md` and repo process docs.
  3. Accepted runtime frame and packet train.
  4. Current OpenSpec packet/workstream records.
  5. Current code, tests, generated-output status, and local commands.
  6. Merged Git history and Graphite state.
  7. Takeover transcript as discovery material only.
- Minimum corroboration: current file evidence plus either command output, packet ledger, or reviewer confirmation for material claims.
- Conflict rule: current code and current OpenSpec/proof records outrank transcript summaries; accepted project authority outranks old branch notes; runtime behavior claims require runtime proof records.
- Claim-strength labels allowed: verified, corroborated, plausible, unresolved, historical.
- Required uncertainty markers: not claimed, not run, historical, current as of main, requires final drain, outside this prework.
- What does not count as evidence: model memory, stale session tail without current file confirmation, green local tests as live product proof, submitted PRs as behavior proof, historical packet text as current implementation state.
- Code-vs-doc rule: code/tests prove current implementation shape; docs/OpenSpec prove target/authority/proof claims; conflicts require explicit classification.
