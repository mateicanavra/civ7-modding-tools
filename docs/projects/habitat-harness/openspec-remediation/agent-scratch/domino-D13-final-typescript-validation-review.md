## Review Findings

### State-space collapse

No unresolved P1/P2.

The repaired packet collapses the relevant design state before writes:

- `SupportedProjectKind` is closed to `plugin | foundation | app`.
- Raw generator options are required to parse into a closed `ScaffoldingRequest`.
- `ScaffoldingDecision` is closed to project write, pattern candidate write, registered-promotion route, or refusal.
- Only write decisions can carry a non-empty write set.
- Refusal decisions carry an empty write set and cannot call write helpers.
- Dispatch over `ScaffoldingDecision` must be exhaustive.

This is the correct TypeScript/refactoring target shape for later implementation: parse at the boundary, model the state machine as a discriminated union or equivalent closed decision type, make illegal write/refusal combinations unrepresentable, and require exhaustive handling.
