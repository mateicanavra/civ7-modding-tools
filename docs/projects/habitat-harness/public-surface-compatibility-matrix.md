## State Glossary

| contract_state | Meaning |
| --- | --- |
| public-stable | Durable public behavior that later packets must preserve unless a packet explicitly changes it. |
| public-versioned | Public surface that may evolve only through an explicit compatibility/versioning path. |
| package-internal | Exported or reachable for package construction, but not a stable public product promise. |
| command-only-dto | Stable for command JSON/receipt consumers, not necessarily for arbitrary library importers. |
| test-only | Exists to support tests/probes and must not become product API by accident. |
| generated-derived | Derived from authoritative generated or registry data; update through the generator/registry authority. |
| deprecated | Historical or discouraged surface retained only for compatibility or migration clarity. |
| refused | Unsupported surface explicitly refused or documented as not current behavior. |
| docs-example | Documentation citation that cannot define behavior without a non-doc row. |

| compatibility_handling | Meaning |
| --- | --- |
| preserve | Keep behavior stable. |
| version | Change only through explicit version/compatibility handling. |
| facade | Keep a compatibility facade while internals can move. |
| deprecate | Keep visible only as deprecated or migration surface. |
| refuse | Refuse unsupported or superseded input explicitly. |
| document-only | Documentation row; behavior belongs to linked non-doc rows. |
| generated-only | Regenerate from authority; do not hand-edit downstream output. |

| row_relationship | Meaning |
| --- | --- |
| same-surface-other-plane | Same durable behavior viewed through another plane. |
| renamed-from | Current row replaces an older row/surface name. |
| renamed-to | Deprecated row points to replacement. |
| deprecated-replacement | Replacement for deprecated surface. |
| generated-from | Row is generated from an authority file. |
| docs-example-of | Documentation example of a non-doc row. |

Every `row_relationships` cell is a typed list of `{"relation":"<row_relationship>","surface_id":"<D0 row id>"}` entries. Empty relationship sets are `[]`. Free-form `relation:surface_id` strings are intentionally not accepted because downstream packets need parseable citations.

## Plane Authority Rules

- `cli`: command syntax and flags are the command surface. JSON schemas and human wording live in their own planes.
- `command-json`: command JSON and receipt DTOs are citeable by D1/D12 only through rows in this plane.
- `durable-data`: repo-local durable data files are citeable by later packets only through rows in this plane. Package allowlist rows do not define the contained file contract.
- `human-output`: terminal text may guide humans, but parsing must cite `command-json` rows.
- `package-export`: package subpaths, manifest allowlists, package scripts, and `src/index.ts` symbols are visible surfaces, but each row declares whether it is public, command-only, internal, generated, or test-only.
- `root-script`: root `package.json` scripts are public repo-local command affordances and are distinct from direct CLI invocations.
- `nx-target`: explicit and inferred Nx targets are graph surfaces. Target existence depends on dependency/build grounding; a missing local dependency does not prove a target absent.
- `generator`: generators must preserve schemas and refusal behavior. Candidate output is not automatically a registered rule.
- `migration`: migrations are durable package surfaces but do not imply source behavior until executed.
- `hook`: hooks are workstation hook check only. Hook pass is not CI, packet closure, or product approval.
- `docs-example`: docs examples cite behavior; they do not create behavior. Stale examples are represented as deprecated/refused rows or as docs rows pointing at refused rows.
