## Acceptance Criteria

- Run in Game is separate from preview/browser Run.
- Studio can prepare/start a fresh single-player game from shell or an existing
  game using the selected/current Studio config, map row, map size/options, and
  Studio seed.
- Seed stays outside authored Swooper config and is applied through Civ setup.
- Materialization policy distinguishes durable Save/Run from disposable
  `studio-current` behavior.
- `@civ7/direct-control` owns setup/start wrappers; Studio never sends raw setup
  JavaScript directly.
- Proof includes `GameInfo.Maps` row existence, setup seed applied, post-start
  `GameplayMap.getRandomSeed()` match, and Swooper log/config hash evidence for
  the exact config loaded.
- Reload semantics is a gating discovery item before claiming exact current
  config behavior.
- Studio live runtime state is observational and never auto-writes
  `pipelineConfig`; runtime-to-config translation is explicit suggestions only.
