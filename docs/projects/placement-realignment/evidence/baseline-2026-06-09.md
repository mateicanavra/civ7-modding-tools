# Placement Baseline — 2026-06-09 (S0 metrics harness)

Instrumented baseline for the placement-realignment workstream, recorded by the
S0 metrics harness BEFORE any behavior change. All later slices are measured
against these numbers. Expectation IDs and predeclared ranges:
`docs/projects/placement-realignment/expectations.md`. Defect predictions:
`docs/projects/placement-realignment/diagnosis.md`.

## Setup

- Harness: `scripts/placement/placement-metrics.ts` (module:
  `mods/mod-swooper-maps/src/dev/diagnostics/placement-metrics.ts`)
- Pipeline: standard recipe, `swooper-earthlike` shipped config, mock adapter,
  headless (same compile+run path as the studio browser worker)
- Map: standard (84x54), 8 players, hemisphere split `PlayersLandmass1=4`,
  `PlayersLandmass2=4`, latitudes -60..60, 4x4 start sectors
- Seeds: 1337..1341 (5 seeds), plus one studio-mapinfo probe (seed 1337)
- Commit context: branch `placement-realignment`, harness added by the S0 change

## Reproduction

```sh
# 5-seed standard baseline (aggregate JSON on stdout, full runs via --json)
bun run verify:placement-metrics -- --seed 1337 --seeds 5 --size standard --json /tmp/pm-baseline.json

# studio-worker doubling probe (PlayersLandmass1 = PlayersLandmass2 = 8)
bun run verify:placement-metrics -- --seed 1337 --seeds 1 --size standard --studio-mapinfo

# harness self-test
bun --cwd mods/mod-swooper-maps test test/placement/placement-metrics-harness.test.ts
```

Runs are deterministic per seed (asserted by the harness self-test).

Key mechanism confirmed: counts are conserved through the pipeline
(planned == assigned == final) while **~98% of placements change type** between
plan and final surface, with only ~17% of preferred types legal at their plot.
Distribution "works" by abandoning identity — exactly the diagnosis RC
(identity-blind planner + force-equalizing materializer + legality rescue).

## Confirmed defects (vs diagnosis predictions)

1. **Start doubling in studio-like runs (E1.2)** — CONFIRMED: with
   `PlayersLandmass1 = PlayersLandmass2 = 8` (the studio worker's mapInfo
   shape), 16 starts are seated for 8 intended players, every one
   `regionalAssigned` (i.e. not a fallback artifact). Fix target: S1.
2. **No impassability screen for starts (E1.1)** — CONFIRMED: mountain and
   volcano starts on 2/5 seeds. Fix target: S1.
3. **Near-uniform per-type resource counts (E2.1)** — CONFIRMED: CV ~0.08
   across 34 types (3–6 placed each); rarity stratification absent. Fix
   target: S3 (after S2 restores Weight).
4. **Legality-driven type reassignment (E2.9)** — CONFIRMED at ~98%.
   Fix target: S2 (plan within policy) + S3 (typed reconcile).
5. **No start support floor/equity (E3.1–E3.3)** — CONFIRMED: floor violated
   on 3/5 seeds; equity gap 7–9 vs <= 2. Fix target: S5.
6. **Start quality claims (E1.4)** — starts are fertility-neutral (~1.05x),
   contradicting "scoring uses fertility" being sufficient; selection-time
   degradation is measurable. Fix target: S4.

## Predictions NOT reproduced (recorded, not hidden)

- **Marine resources (E2.4):** mock-adapter runs place 20–25 water resources.
  The "suspected 0" came from live-run observation; the live half stays open
  for Milestone A. If live also places marine resources, amend expectations.
- **Fallback usage (E1.7):** 0% openPool/desperation on these 5 seeds at
  standard size with the earthlike config. The silent-fallback *mechanism*
  still exists (diagnosis RC); it just doesn't fire here. E1.7's 20-seed
  window stays the gate.
- **Spacing decay to 0 (E2.6):** global same-type min distance never fell
  below the planned floor (2) on these seeds. The decay path remains in the
  materializer rescue logic; RDP step 2 (S1d) still owes the fix.

## Harness scope notes

- All metrics labeled `pending-s2/s3/s4` or `requires-live-engine/studio-dump`
  are reported with explicit status in the JSON output, never omitted.
- `preferredLegalityRate` uses the mock adapter's static policy emulation
  (`Resource_ValidBiomes` tables) evaluated on the final surface; mock-vs-live
  agreement is bounded separately by E4.4 at Milestone A.
- The harness REPORTS only. Nothing gates on these ranges until a later slice
  flips a metric to a gate with this baseline as evidence.
