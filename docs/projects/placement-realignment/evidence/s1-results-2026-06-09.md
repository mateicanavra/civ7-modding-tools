## What S1 changed (mechanisms, not tuning)

1. **S1a (E1.1):** plan-starts admission now rejects mountain
   (`morphology.mountains.mountainMask`), volcano
   (`morphology.volcanoes.volcanoMask`), and natural-wonder-occupied tiles
   (placed rows of `placement.naturalWonderPlacement`, incl. observed anchor +
   expected footprint plots) with typed rejection reasons
   `mountain|volcano|natural-wonder`; the assign-starts step contract now
   requires those three artifacts. The desperation fallback is screened by the
   same exclusion mask so the invariant survives even when planned candidates
   run out.
2. **S1b (E1.2 studio half):** `pipeline.worker.ts` no longer duplicates
   `playerCount` into both `PlayersLandmass` slots; it splits
   ceil/floor per base-game per-hemisphere semantics.
3. **S1c (E1.7):** openPool/desperation seating emits `console.warn` plus a
   warn-tagged trace event (`placement.starts.fallback`), and the
   `startAssignment` artifact records per-seat `seatPaths`
   (`regional|openPool|desperation`). Selection behavior unchanged.
4. **S1d (RDP step 2):** the relaxed-spacing assignment phase (spacing decay
   to 0) is removed; unmeetable target counts are recorded as
   `spacingShortfallCount` in the `resourcePlacementOutcomes.assignment`
   summary + runtime telemetry + a warn trace event
   (`placement.resources.spacingShortfall`). The whole-map dispersed plot
   fallback itself remains (S3 removes it).
5. **S1e:** dead `chooseLeastUsedLegalResource` deleted (zero callers);
   `findLegalPlotWithRelaxedSpacing` deleted with the relaxed phase.
