## Purpose And Source Boundary

This ledger tracks every live rule currently present in the Habitat authority tree. It is a process-control artifact, not a new ontology and not a replacement for `.habitat/dominoes.md`.

Source boundary:

- Live `.habitat/**/rule.json` manifests enumerate the rule corpus and provide current placement facts.
- `.habitat/dominoes.md` is the only source for process, disposition, bucket, and pending-action evidence.
- Stale domino rows for rules that no longer exist are excluded from the live matrix and listed separately.
- Missing disposition information stays missing; this ledger does not classify artifact, config, projection, domain, or cleanup pressure beyond the domino text.

## Status Semantics

| Process status | Meaning | Pending action rule |
| --- | --- | --- |
| `processed: receipt` | The live rule has an explicit row in a Domino disposition receipt. | Copy the existing `Next trigger`, `Deferred cleanup signal`, `Reusable lesson`, or nearest equivalent from `.habitat/dominoes.md`. |
| `processed: summary-only` | The live rule sits in a completed pocket named by `.habitat/dominoes.md`, but no per-rule receipt row exists. | `not specified in dominoes.md` |
| `not processed in dominoes.md` | The live rule has no specific receipt row and no completed-pocket summary evidence in `.habitat/dominoes.md`. | `still to be processed` |
| `stale domino reference` | A rule appears in `.habitat/dominoes.md` but no longer exists as a live rule. | List only in `Stale Domino References`; do not include in the live matrix. |
